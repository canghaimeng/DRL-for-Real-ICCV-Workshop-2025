Development Phase

The Development Phase of the DRL for Real competition is designed to allow participants to develop and refine their disentangled representation learning methods using the provided training datasets. This phase focuses on model development, experimentation, and validation.

This phase runs from January 15, 2025 to August 15, 2025, providing participants with seven months to develop robust disentangled representation learning methods.

Development Phase Details:

  • Available Resources:
    • Training datasets containing real-world data with various factors of variation
    • Validation datasets for preliminary evaluation
    • Baseline implementations of common disentanglement methods
    • Evaluation code implementing the competition metrics
  • Participant Activities:
    • Develop models that learn disentangled representations of the data factors
    • Experiment with different architectures, loss functions, and training strategies
    • Evaluate models on the validation set using the provided evaluation metrics
    • Receive feedback on model performance in terms of disentanglement, robustness, interpretability, and practical utility
  • Evaluation Criteria:

    Evaluation criteria vary by track:

    • Single-Factor Track:
      • Image Generation Quality (50%): Measured using FID
      • Disentanglement Quality (50%): Evaluated using Bi-directional DCI
    • Multi-Factor Track - General Image Dataset:
      • Main Leaderboard: LLM+VQA evaluation approach
      • Disentanglement Leaderboard: FID (50%) and Bi-directional DCI (50%)
    • Multi-Factor Track - Autonomous Driving Dataset:
      • Main Leaderboard: SSIM between generated images and ground truth
      • Disentanglement Leaderboard: FID (50%) and Bi-directional DCI (50%)

The leaderboard for this phase will not be public, allowing participants to experiment with different approaches without revealing their performance to other competitors. This encourages exploration and innovation without the pressure of public rankings.

Submission Guidelines:

  • Participants can make up to 5 submissions per day
  • Maximum of 50 submissions in total during this phase
  • Each submission should include model code and any necessary weights
  • A brief description of the approach is encouraged but not required

We encourage participants to focus on developing methods that not only achieve high disentanglement scores but also demonstrate practical utility in real-world applications. The most successful approaches will balance theoretical disentanglement properties with practical considerations such as computational efficiency and downstream task performance.
